您可以把一个任务的历史检查导出为CSV文件，这对调试和审计很有用。  
您可以导出一个任务的最近1000次历史执行记录，每个任务每15分钟限导出一次。 

您最好使用Chrome浏览器来导出历史记录并下载CSV文件，其他浏览器可能不支持这个功能。  

## CSV文件中的列  

_id: 此次检测对应历史记录的id.  
scheduledTime: 原计划执行时间.(时间戳，毫秒)  
consumeTime: 分布式浏览器接收到此次任务的时间.(时间戳，毫秒)  
**finishTime**: 完成此次检测任务的时间。(UTC时间，时区为UTC+0)   
**err**: 分布式浏览器的错误信息.  
textHash: 网页区域文本的哈希值.  
**textContent**: 网页区域的文本。(如果超过2000个字符，将被截断)。  
taskId: 此次检测属于的任务 的ID。  
最后一列: 此结果是否已被任务检查器检查过.  

:::tip
您需要关注 **finishTime**, (UTC时间，时区为UTC+0).  
您需要关注 **err**, 如果一次检测失败了，它将显示一些原因.  
您需要关注 **textContent**, 观察该内容是否是您要监测的正确文本. （使用CSS选择器选择您关心的区域的文本）.
:::
